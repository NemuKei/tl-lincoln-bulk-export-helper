# Purpose
タスク追加のたびにバックログを再整理し、重複や順序崩れを防ぐ。
実装時に迷わない粒度へ正規化する。
root `AGENTS.md` の repo-wide ルールを前提にし、この Skill は task add and triage に必要な task-specific procedure だけを追加する。

# When to use / When NOT to use
- When to use:
  - `docs/tasks_backlog.md` に新規タスクを追加した直後
  - 追加タスクが既存タスクと重複・依存しそうなとき
  - 実装順序を `Now/Next` に反映したいとき
- When NOT to use:
  - 単純な typo 修正のみでタスク構造が変わらないとき
  - 完了チェックの更新だけで、優先順位見直しが不要なとき

# Inputs
- 追加したタスク
- 未実装タスク一覧
- 粒度判定に必要な変更面

# Outputs
- 棚卸し結果
- 必要時は分割済みタスク
- 最適化後の未実装タスク実行順

# Procedure
1. 新規タスク追加後、未実装タスクを一覧化する。
2. 追加タスクの粒度を判定する。
3. 粒度が粗い場合は子タスクへ分割する。
4. 重複・包含関係を確認する。
5. 依存関係と効果対コストに基づいて順番を最適化する。
6. 更新後、どこまで実装対象かをユーザーへ明示する。

# Validation
- 新規追加タスクが実装粒度として過不足ない。
- 重複・依存・優先度の判断理由がある。
- 実装順が現在のバックログ構造と整合している。
