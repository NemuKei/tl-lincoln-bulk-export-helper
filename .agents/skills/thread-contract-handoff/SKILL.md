# Purpose
1 スレッドで扱う目的と範囲を明確にし、途中で出た別論点を無秩序に混ぜず、必要な場合だけ次スレッド用 handoff へ整理する。

# When to use / When NOT to use
- When to use:
  - handoff prompt から新しいスレッドを始めるとき
  - 実装、仕様壁打ち、レビュー、調査など複数段階になりうるスレッドを始めるとき
  - 話題が広がりやすく、どこまでこのスレッドで扱うかを先に固定したいとき
- When NOT to use:
  - 単発の短い質問で、その場で回答して終わるとき
  - 1 往復で閉じられる軽微な依頼のとき

# Thread contract
スレッド開始時に、少なくとも次を短く明示する。

- `このスレッドの目的`
- `今回の範囲`
- `今回やらないこと`
- `終了条件`
- `終了時に確認が必要なもの`

# Procedure
1. スレッド冒頭で依頼内容を読み、今回の主目的を 1 つに定める。
2. `今回の範囲` と `今回やらないこと` を切り分ける。
3. `終了条件` を、作業完了条件と確認条件に分けて定義する。
4. 実行中に新しい論点が出たら、現在の主目的に必要かどうかで現在スレッド扱いか parking かを分ける。
5. handoff prompt は自動で出さず、終了時に必ずユーザー確認を入れる。

# Validation
- スレッド冒頭で `目的 / 範囲 / やらないこと / 終了条件` が明示されている。
- 途中で広がった論点が parking されている。
- handoff prompt を出す前にユーザー確認を入れている。
