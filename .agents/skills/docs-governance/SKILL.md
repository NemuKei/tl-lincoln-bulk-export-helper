# Purpose
ドキュメント運用を重複なく維持し、正本への反映をファイル更新で確定する。
root `AGENTS.md` の repo-wide ルールを前提にし、この Skill は docs governance に必要な task-specific procedure だけを追加する。

# When to use / When NOT to use
- When to use:
  - 新規ドキュメントを作るか判断するとき
  - 同一ルールの重複を整理するとき
  - 壁打ち会話の内容を正本へ反映するとき
- When NOT to use:
  - 単なる草案共有で、正本反映をまだ決めていないとき
  - 新規作成の3条件を満たせないとき

# Inputs
- 追加・更新したいルールや仕様
- 既存ドキュメント群（責務分離の可否判断に必要な範囲）
- 所有者と更新トリガーを定義できる情報

# Outputs
- 新規作成または更新された正本ドキュメント
- 重複を除去した短いリダイレクト文（必要時）

# Procedure
1. 変更内容を既存ドキュメントへ責務分離できるか確認する。
2. 新規ドキュメントが必要な場合は、次の3条件をすべて確認する。
   1. 既存ドキュメントに責務分離できない。
   2. 今後2回以上参照する見込みがある。
   3. 所有者と更新トリガーを定義できる。
3. 3条件を満たさない場合は新規作成しない。既存ドキュメントへ統合する。
4. 同一ルールが複数箇所にある場合、正本を1箇所に定める。重複箇所は短いリダイレクト文へ置換する。
5. 壁打ち会話は非正本として扱い、正本化する場合は必ずファイルを更新して確定する。

# Validation
- 新規作成時は3条件をすべて満たしている。
- 所有者と更新トリガーが定義されている。
- 同一ルールの重複記載が残っていない。
- 重複箇所は短いリダイレクト文へ置換されている。
- 正本反映は会話ではなくファイル更新で確定している。
